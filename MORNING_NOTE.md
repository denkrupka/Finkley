# MORNING NOTE — ночь 20→21 мая 2026

## ✅ Сделано и задеплоено (можно тестировать)

### Booksy интеграция

- **Длительность визитов** — теперь синкается из `booked_till - booked_from`. Раньше всегда было 60 мин.
- **«Rezerwacja czasu»** — резервы мастеров импортируются в портал как штрихованные блоки.
- **Кнопка «Синхронизировать»** на странице визитов — синкает текущий день из URL. При смене дня в календаре автосинк через 600мс (debounce + rate-limit 8с).
- **Reverse-sync создания**: визит в портале → создаётся reservation в Booksy; блок в портале → reservation в Booksy.
- **Reverse-sync удаления**: визит/блок удалили в портале → отменяется в Booksy через корректный endpoint `POST /appointments/{uid}/action/ {action:"cancel", _version}`. Старый DELETE возвращал 302 на главную и молча игнорировался — исправлено.
- **Reverse-delete**: визит/блок удалили в Booksy → пропадёт в портале на следующем sync (визиты только будущие, блоки все).
- **Move в Booksy** → обновляется visit_at в портале (раньше ignoreDuplicates пропускал).
- **Paid/Pending статус** — теперь определяется по наличию `basket_id` в Booksy appointment (не по status='F'). Pending→Paid апгрейд работает на следующем sync, обратно не делаем.
- **portal-owned reservations** не импортируются обратно (image #20 — больше не накладывается блок поверх своего же визита).
- **Cancelled (status=C)** резервации пропускаются при импорте.

### KSeF

- **Подключение чинено** — KSeF API сменил формат `timestamp` на ISO. Стало parsing'ом NaN → invalid_timestamp. Теперь предпочитаем `timestampMs` поле, fallback на парсинг ISO.
- **Async auth** — между `POST /auth/ksef-token` (202) и `/auth/token/redeem` (400) добавлен polling статуса `GET /auth/{refNum}` до status=200. Раньше валилось.
- **Invoice query shape** — `subjectType` + `dateRange` теперь на корне body (не в `invoiceQueryFilters`). Response field — `invoices`, item-поля: `ksefNumber`, `seller.nip`, `buyer.identifier.value`, `grossAmount`. Всё переписано.
- **E2E проверено напрямую с твоим токеном** (NIP 7831854263) — получил 12+ реальных фактур включая Culligan, заносится в expenses.

### Отчёты

- **ДДС**: показывает только paid визиты + actual expenses. Pending визиты не попадают.
- **Финансовый отчёт**:
  - План = все визиты (paid+pending) + expenses + scheduled_payments.
  - Факт = только paid визиты + actual expenses.
  - В колонке Расходы теперь видны реальные суммы из БД (раньше было пусто если variable/fixed/taxes не настроены).
- **Conflict-check визитов** — теперь игнорирует soft-deleted (раньше «У мастера уже есть визит» вылезало после удаления).

## ⚠ Что важно протестировать первым делом

1. **KSeF** — нажми «Подключить» с твоим токеном. Должно пройти. Потом «Синхронизировать» — должны прилететь фактуры в Расходы.
2. **Booksy paid статус** — нажми «Синхронизировать» на странице визитов. Визиты с зелёным $ в Booksy должны стать «оплачено» в портале.
3. **Move в Booksy** — перенеси визит на другой день в Booksy, нажми «Синхронизировать» — должен переехать в портале.
4. **Финансовый отчёт** — открой Финансы → Финансовый отчёт. В Расходах должны быть числа.
5. **ДДС** — открой Финансы → ДДС. Должны быть только paid визиты.

## 🐛 Известные ограничения

- **Существующие визиты до фикса paid-статуса** не апдейтятся автоматически. Поможет нажать «Синхронизировать» на нужный день — pending→paid апгрейд произойдёт.
- **Legacy блоки/визиты** созданные до фикса с `external_reservation_id=null` — их удаление в портале не каскадит в Booksy. Один раз убери в Booksy руками.
- **Booksy reservations stuck** — несколько резерваций c старыми uid могут остаться. После следующего sync они либо отменятся либо появятся в портале.
