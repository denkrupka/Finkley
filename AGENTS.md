# AGENTS.md — Как работать с Claude Code

> Этот документ — про **тебя**, владельца проекта. Не про Claude Code как инструмент, а про твою стратегию использования.

## Зачем этот документ

Соло-разработчик с Claude Code — это не "запустил и пошёл пить кофе на неделю". Это **новая профессия**, которую ты осваиваешь. Этот документ — твои опорные правила, чтобы не сделать типичных ошибок.

Прочитай раз перед первой сессией. Перечитывай раз в месяц.

## Реалистичная картина

### Что Claude Code делает быстро и хорошо

- Boilerplate (CRUD-формы, списки, простые компоненты)
- Применение паттернов из существующего кода
- Тесты для существующего кода
- Миграции БД по описанию
- Рефакторинг механический (rename, extract function)
- Документация по коду

**Реальная скорость:** в 3–10 раз быстрее тебя.

### Что Claude Code делает средне

- UX-решения (какую кнопку поставить, как сгруппировать поля)
- Дизайн состояний (loading, error, empty)
- Сложные SQL-запросы с агрегациями
- Дебаг неочевидных багов

**Скорость:** примерно как у тебя, но без выгорания.

### Что Claude Code делает плохо

- Продуктовые решения ("надо ли вообще делать эту фичу?")
- Архитектурные решения на стыке стадий
- Решения с финансовыми последствиями
- Юр.вопросы (Privacy, Terms, лицензии)
- Маркетинг и тон коммуникации с пользователями
- Балансировка приоритетов

**Это твоя работа.** Не делегируй.

## Распределение ролей

| Что делаешь ты                 | Что делает Claude Code         |
| ------------------------------ | ------------------------------ |
| Решаешь "что строим"           | Делает "как строим"            |
| Решения с пользователями       | Реализация решений             |
| Talk to customers              | Implement features             |
| Планирование стадий            | Атомарные задачи внутри стадии |
| ADR на крупные решения         | ADR на технические детали      |
| Маркетинг, копирайт, продажи   | Копирайт UI через i18n         |
| Code review (выборочно)        | Self-review через DoD          |
| Rollback при критических багах | Bug fixes по тикетам           |
| Stripe blocked, GDPR incident  | Routine support tasks          |

## Режимы работы

### 🟢 Автономный режим

Claude Code работает сам, ты периодически проверяешь.

**Когда:**

- Задача атомарная (TASK с чёткими AC)
- Логика тривиальная (CRUD, отрисовка)
- Есть паттерн в репо для подражания

**Ты:**

- Запускаешь промпт → уходишь на 30-60 минут → возвращаешься → проверяешь PR

**Подходит для:** TASK-11 (список визитов), TASK-13 (расходы), TASK-12 (мастера и услуги).

### 🟡 Соавторский режим

Claude Code пишет, ты сидишь рядом и направляешь.

**Когда:**

- Задача нетривиальная (XL или с развилками)
- Много развилок где нужно выбирать вариант
- Сложная UX-логика
- Интеграции с внешними сервисами

**Ты:** сидишь рядом 1-2 часа, отвечаешь на вопросы, делаешь UX-решения, контролируешь архитектуру.

**Подходит для:** TASK-08 (онбординг), TASK-14 (дашборд), TASK-16 (Stripe), TASK-22 (расчёт зарплат).

### 🔴 Веду сам, Claude помогает

Ты пишешь основу, Claude дополняет.

**Когда:**

- Критическая бизнес-логика (расчёт прибыли, payouts)
- Sensitive UX (онбординг, момент платежа)
- Реакция на реальный фидбек первых пользователей

**Claude делает:** тесты к твоему коду, boilerplate вокруг твоей логики, документация.

## Анти-паттерны

### ❌ "Запустил и забыл на неделю"

Claude Code пишет 4-8 часов автономно, потом нужна ревизия. На длинных задачах **обязательно** есть моменты выбора, неоднозначности, упущенные требования.

**Правильно:** check-in каждые 1-3 часа.

### ❌ Принимать всё что он пишет без ревью

Claude Code пишет код, который **компилируется и проходит тесты**. Это не значит что код **правильный**. Особенно опасно:

- Правки RLS-политик (могут открыть утечку)
- Изменения схемы БД (могут потерять данные)
- Изменения в auth flow
- Биллинг-логика (Stripe webhook handling)

**Правильно:** на критичных файлах — ручное ревью построчно.

### ❌ Ругать или давить

Не работает быстрее от мата. Контекст загрязняется, качество падает.

**Правильно:** конкретные claims. "Этот тест падает потому что X. Почему?"

### ❌ Длинные сессии без перерывов

После 4-6 часов в одной сессии Claude Code теряет фокус, начинает повторяться.

**Правильно:** короткие сессии (1-2 часа), коммит между ними, при необходимости — рестарт.

### ❌ Игнорировать ADR

"Это мелкое решение, без ADR обойдусь". Через 2 месяца никто не помнит почему.

**Правильно:** любое решение которое требует объяснения "почему" — в ADR.

### ❌ Делегировать продуктовые решения

"Claude, что лучше — €15 или €19?" Это **твой** вопрос. У Claude нет понимания твоих пользователей.

**Правильно:** Claude помогает с **технической стороной**, не с бизнес-выбором.

### ❌ Запускать Claude Code на проде

Никогда. Он работает с staging БД. На прод — деплой через GitHub Actions после твоего ревью.

## Когда Claude Code не справляется

Признаки что задача **не для Claude Code**:

1. Описание требует разговора с реальным пользователем
2. Решение зависит от субъективных бизнес-предпочтений
3. Задача про маркетинг или стратегию
4. Требует креативности на уровне "идеи продукта"

**Что делать:** делай сам или нанимай человека (фриланс-дизайнер, фриланс-маркетолог).

## Использование sub-agents

Если в твоей версии Claude Code работают sub-agents (см. `.claude/agents/README.md`):

- **architect** — обсудить архитектурное решение, написать ADR (без кода)
- **migrator** — только SQL-миграции БД
- **reviewer** — холодный взгляд на готовый PR
- **tester** — написать тесты для существующего кода

Запрос главному агенту: «передай задачу sub-agent X». Главный делегирует, ты получаешь результат в чистом контексте.

**Когда полезно:** на длинных задачах когда контекст загрязнён. **Когда не нужно:** на простых задачах — лишний overhead.

## Финансовая гигиена

Anthropic API стоит денег. Ориентир расходов:

- **Соло-разработчик**, активно: $100-300/мес на API
- **Если упрёшься в лимит** — Claude Code зависнет посреди задачи

**Совет:** держи буфер +50% к ожидаемому расходу. Лучше переплатить $50, чем потерять день дебага "почему перестал отвечать".

## Этический аспект

Ты — единственный человек в проекте. Это значит:

- Ты несёшь полную ответственность за продукт
- Не вали баги на "это Claude Code написал"
- Не пиши пользователям "AI допустил ошибку" — это твоя ответственность
- Если код плохой — это потому что ты не проревьюил

Claude Code — инструмент, как Excel или Photoshop. Ответственность за результат — твоя.
