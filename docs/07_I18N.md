# 07. Internationalization (i18n)

## Стратегия

- **MVP:** только RU
- **V1+:** PL, EN, UK по мере выхода на рынки

Принцип: **с дня 1 всё через i18n, никакого хардкода.** Добавление нового языка = добавление JSON.

## Стек

- **react-i18next** + **i18next** + **i18next-browser-languagedetector**

Установка:

```bash
pnpm --filter web add react-i18next i18next i18next-browser-languagedetector
```

## Структура

```
apps/web/src/i18n/
├── index.ts
└── locales/
    ├── ru.json
    ├── pl.json   # стадия 5
    └── en.json   # стадия 5
```

`apps/web/src/i18n/index.ts`:

```typescript
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import LanguageDetector from 'i18next-browser-languagedetector'
import ru from './locales/ru.json'

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: { ru: { translation: ru } },
    fallbackLng: 'ru',
    supportedLngs: ['ru'],
    interpolation: { escapeValue: false },
    detection: { order: ['localStorage', 'navigator'], caches: ['localStorage'] },
  })

export default i18n
```

В `main.tsx`:

```typescript
import './i18n'
```

## Использование

```tsx
import { useTranslation } from 'react-i18next'

export function LoginForm() {
  const { t } = useTranslation()
  return (
    <form>
      <h1>{t('auth.login.title')}</h1>
      <button>{t('auth.login.submit')}</button>
    </form>
  )
}
```

## Форматирование чисел и дат — НЕ через i18next

Используем нативный `Intl` и `date-fns`:

```typescript
// lib/utils/format-currency.ts
export function formatCurrency(cents: number, currency = 'PLN', locale = 'ru'): string {
  return new Intl.NumberFormat(locale, { style: 'currency', currency }).format(cents / 100)
}

// lib/utils/format-date.ts
import { format, formatRelative } from 'date-fns'
import { ru, pl, enUS } from 'date-fns/locale'

const locales = { ru, pl, en: enUS }

export function formatDate(date: Date | string, fmt = 'PPP', locale = 'ru'): string {
  const d = typeof date === 'string' ? new Date(date) : date
  return format(d, fmt, { locale: locales[locale as keyof typeof locales] })
}
```

## Плюрализация

```json
{
  "visits": {
    "count_one": "{{count}} визит",
    "count_few": "{{count}} визита",
    "count_many": "{{count}} визитов"
  }
}
```

```tsx
{
  t('visits.count', { count: 5 })
} // "5 визитов"
```

i18next понимает русские правила плюрализации из коробки.

## Что переводим

- Все строки UI
- Email-шаблоны (Postmark templates на каждый язык)
- Privacy Policy, Terms (через юриста)
- Лендинг (apps/landing)

## Что НЕ переводим

- Имена категорий, услуг, мастеров, клиентов (вводит юзер)
- Имена брендов

## Активный язык

В стадии 1 — только RU.

В стадии 5 при добавлении PL/EN/UK:

- Переключатель в шапке
- Сохранение в `profiles.locale`
- При логине — `i18n.changeLanguage(profile.locale)`

## CI-проверка ключей

`scripts/check-i18n-keys.ts`:

- Парсит `apps/web/src/**/*.tsx`, ищет `t('xxx')`
- Сверяет с `locales/ru.json`
- Фейлит CI если ключ не найден

В стадии 5 — добавить проверку, что все ключи RU есть и в PL/EN.

## Качество переводов (стадия 5)

- Native speaker review (фриланс ~$0.10/слово)
- Скриншоты UI → проверка длины строк (PL/DE часто длиннее RU)
