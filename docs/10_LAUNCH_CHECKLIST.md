# 10. Launch Checklist

Чек-лист до публичного запуска.

## Перед закрытой бетой (конец стадии 1)

### Технический

- [ ] Все TASK-XX из стадии 1 выполнены
- [ ] RLS-политики покрыты тестами
- [ ] `pnpm typecheck`, `pnpm lint`, `pnpm test`, `pnpm build` зелёные
- [ ] Production Supabase, миграции применены
- [ ] GitHub Pages production деплой работает
- [ ] HTTPS активен, кастомный домен настроен
- [ ] SPA routing fallback (404.html) работает
- [ ] Smoke тест на iPhone Safari, Android Chrome, Desktop Firefox
- [ ] Sentry получает события
- [ ] Postmark отправляет welcome email
- [ ] Stripe в test mode принимает платежи

### Контент и юр

- [ ] Лендинг (минимум: hero, features, FAQ, CTA, footer)
- [ ] Privacy Policy и Terms опубликованы
- [ ] `support@finkley.app`, `privacy@finkley.app` принимают почту
- [ ] PL JDG активно, NIP получен
- [ ] DPA подписаны: Supabase, Postmark, Anthropic, Sentry, Stripe

### Бета-тестеры

- [ ] Список 5–10 знакомых владелиц
- [ ] Канал общения для фидбека (Telegram-чат)
- [ ] Краткий гайд "как пользоваться" (2 страницы)
- [ ] Stripe coupon 100% off на 6 месяцев для бетеров

## Перед публичным запуском (конец стадии 2)

### Технический

- [ ] Все TASK-XX из стадии 2 выполнены
- [ ] RLS-тесты на CI
- [ ] E2E Playwright: signup, onboarding, ввод визита, ввод расхода, дашборд, оформление подписки
- [ ] Дашборд за месяц грузится <2 сек на 3G
- [ ] Lighthouse: Performance ≥80, Accessibility ≥90, Best Practices ≥90
- [ ] Stripe в live mode, тестовый платёж реальной картой прошёл
- [ ] Stripe Tax корректно считает VAT для PL/EU
- [ ] Все cron работают на staging
- [ ] Backup тестировался (восстановление из бэкапа на staging)
- [ ] Incident runbook (`docs/runbook.md`)
- [ ] Domain зарегистрирован минимум на 2 года
- [ ] DKIM/SPF/DMARC валидны (mail-tester.com ≥9/10)

### Безопасность

- [ ] `SECRETS_ENCRYPTION_KEY` в Supabase Function secrets
- [ ] gitleaks pre-commit hook
- [ ] GitHub Secret Scanning включён
- [ ] Все секреты ротированы перед запуском
- [ ] CSP заголовки активны
- [ ] HSTS активен

### Контент

- [ ] Лендинг с реальными скриншотами
- [ ] Privacy Policy и Terms ревью юристом (~€100–200)
- [ ] FAQ страница (10–15 ответов)
- [ ] Email шаблоны Postmark финальные на RU
- [ ] Help-центр (`/help`)

### Маркетинг

- [ ] Соц.сети: Instagram, Telegram-канал
- [ ] 5–10 постов готовы к публикации
- [ ] Видео-демо 60 секунд
- [ ] Список 50+ владелиц салонов в PL для cold outreach

### Поддержка

- [ ] Канал поддержки: email + Telegram
- [ ] SLA: ответ в 24 часа в рабочие дни
- [ ] Шаблоны ответов на FAQ
- [ ] Понимание, кто отвечает в первый месяц (ты сам)

### Метрики

- [ ] Goatcounter/Plausible работает, цели: signup, first_visit_added, paid_conversion
- [ ] Sentry alerts на email при критических ошибках
- [ ] Stripe alerts на failed payments
- [ ] Раз в неделю — обзор метрик

## Перед стадией 3 (интеграции)

- [ ] 50+ платящих юзеров (PMF)
- [ ] Booksy research-спайк (TASK-27) завершён
- [ ] ADR-005 финализирован с конкретным выбором
- [ ] OCR Anthropic тестировался на 100+ реальных чеках, точность ≥85%
- [ ] wFirma — 5+ PL-юзеров готовы тестировать

## Перед стадией 4 (рост)

- [ ] 200+ платящих
- [ ] Достаточно данных для бенчмарков (≥10 салонов в группе)
- [ ] AI-инсайты тестировались на закрытой группе
- [ ] PWA тестировался на iOS и Android

## Перед стадией 5 (масштаб)

- [ ] 500+ платящих
- [ ] Несколько мульти-салонных аккаунтов
- [ ] Запросы на роли и приглашения от платящих
- [ ] Время рассмотреть Pro-тариф €29–49/мес

## Что не делать

- ❌ Не обещай функции, которых нет
- ❌ Не запускай в августе/декабре (низкий сезон бьюти)
- ❌ Не запускай в день, когда не сможешь быть на связи 8 часов
- ❌ Не давай скидки 50%+ "на старте"
- ❌ Не пиши "100% безопасно"
- ❌ Не используй стоковые фото с пластиковыми улыбками

## Первая неделя после запуска

- [ ] Каждый день — Sentry, Postmark Activity, Stripe failed payments
- [ ] Каждый день — ответить на support в течение 4 часов
- [ ] Раз в день — short status update в Telegram-канал
- [ ] Раз в неделю — публичная метрика
- [ ] В пятницу — личный ретро в `docs/RETRO.md`

## Что делать, если что-то сломалось

1. **Stripe не принимает платежи** — отдельная страница "напишем когда починим"
2. **БД лежит** — Supabase status page, twitter с честным апдейтом
3. **Утечка данных** — incident response из `06_SECURITY_PRIVACY.md`
4. **Ты не можешь работать** — прямо в Telegram-канал, дать ETA
