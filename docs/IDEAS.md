# Идеи фич для салонов (приоритизация на завтра)

Список того, что **реально добавит ценность** beauty-салону. Приоритизация по
оси «revenue / retention / operations / wow», эффорт по `S/M/L/XL`.

> S = пол-дня, M = 1-2 дня, L = неделя, XL = 2+ недели

---

## 🔥 Revenue (увеличивают чек / повторные визиты)

### 1. Автонапоминания о записи (Telegram/SMS/Email) `M`

За 24 ч до визита — push клиенту с временем и адресом. Снижает no-show на 30-40%
(стандартный benchmark в индустрии). Использовать существующий мессенджер +
template. Опционально: кнопка «подтвердить» / «перенести».

### 2. Программа лояльности — баллы и скидки `M`

Каждый визит начисляет N баллов. На N-й визит — скидка 20%, на 10-й — бесплатная
услуга по выбору. UI: progress bar в карточке клиента, авто-применение скидки в
QuickEntry если у клиента накопилось.

### 3. Re-engagement рассылки «давно не были» `S`

Авто-фильтр клиентов с последним визитом > 60 дней → bulk-рассылка с промокодом
−15% на следующий визит. Конверсия таких кампаний в beauty ~10-15%. У нас уже
есть Bulk-broadcast UI — нужен фильтр + шаблон.

### 4. Up-sell в QuickEntry: «Добавить продукт» `S`

При сохранении визита — bottom sheet «Клиент брал маску за 80₽?» (на основе
истории покупок этого клиента или популярных add-ons). Поднимает средний чек.

### 5. Прайс-публикация в Instagram `M`

Кнопка «Опубликовать прайс» — генерит красивую картинку с услугами/ценами и
заливает в IG Stories через Instagram Graph API. Раз в неделю напоминание.

### 6. Подарочные сертификаты `M`

Клиент покупает сертификат на сумму → код → при следующем визите код = оплата.
Учёт в финансах: deferred revenue.

## 💛 Retention (удержание клиентов и мастеров)

### 7. Карточка клиента — таймлайн `M`

В клиенте: визиты, расходы клиента, любимые услуги, средний чек, мастер по
умолчанию. Когда клиент пишет в мессенджере — администратор сразу видит
историю и предпочтения.

### 8. Дни рождения клиентов `S`

Поле в карточке + дашборд «У кого ДР на этой неделе» + автоматическое
сообщение-поздравление с подарком/скидкой.

### 9. NPS-опросы после визита `S`

Через 2 ч после визита — авто-сообщение «Как прошёл визит? 1-10?». Если &le;7 —
эскалация владельцу. Если 9-10 — просьба написать отзыв в Google.

### 10. Чёрный список / «осторожно» теги клиентов `S`

Тег «не записывать» / «не любит N мастера» / «всегда опаздывает» — видны в
QuickEntry и календаре.

### 11. KPI мастеров с премиями `M`

Цели мастера (выручка/визитов/upsell) → автоматический расчёт премии в конце
месяца. Прозрачность мотивирует.

## ⚙️ Operations (автоматизация рутины)

### 12. Smart-расписание мастеров `L`

Автоматическая оптимизация: смены, перерывы, отпуска. Календарь видит загрузку
и подсказывает: «У Анны окно 14:00-15:30 — предложить клиенту X?»

### 13. Auto-OCR на email-фактуры `S`

Поставщик шлёт фактуру на info@finkley.app/<salon-slug>@ — auto-OCR, авто-расход
в нужную категорию. Используем уже готовый OCR pipeline. Email через Resend
inbound webhook.

### 14. Шаблоны сообщений `S`

Готовые тексты: «подтверждение записи», «перенос», «пост-визит благодарность»,
«promo». Вставка в один клик из мессенджера, с переменными `{{client_name}}`.

### 15. Auto-расчёт зарплат `M`

В конце месяца автоматический расчёт ФОТ по всем мастерам (фикс + %, аренда
кресла, премии) → один клик «Закрыть месяц» → выплаты записаны в расходы.
**Уже сделано частично** — нужно UI для «закрытие периода».

### 16. Учёт расхода материалов на визит `M`

К услуге привязан материал (краска 50г, лак 5 мл). При сохранении визита
автоматически списывается из inventory. Себестоимость услуги виден сразу.

### 17. Бронирование без админа (booking widget) `XL`

Виджет «забронировать» для встраивания в Instagram bio / website. Клиент сам
выбирает мастера/услугу/время — визит создаётся в Finkley. Реальная
конкуренция с Booksy при low-volume салонах.

## 🤖 AI / Wow

### 18. AI-помощник «Что мне делать сегодня?» `M`

Утром при заходе — карточка «Сегодня: 8 визитов, 3 окна у Анны 14:00-16:00,
2 клиента давно не были — отправь им сообщение?» с готовыми CTA-кнопками.

### 19. AI-генерация постов в Instagram `M`

По фото "до/после" — AI пишет caption + hashtags + предложение услуги. Auto-
загрузка через IG Graph API.

### 20. AI-фото портфолио — фильтры до/после `S`

В карточке визита загрузка двух фото "до/после" → автоматическая склейка,
watermark с логотипом салона, готово для постинга.

### 21. AI-анализ переписки `M`

Сканирование messenger-conversations: подсветка вопросов без ответа &gt; 1 ч,
выделение «горячих» клиентов (готовых записаться), автотриаж по приоритету.

### 22. Voice-input на каждой странице `S`

Кнопка-микрофон → говоришь «расход 200 на краску» → Finkley понимает и создаёт
запись. Идеально для мастеров между визитами.

## 🎯 Quick wins (small effort, high impact)

### 23. Темы оформления (light/dark/cream) `S`

**Уже сделано** — light/dark/system. Можно добавить custom-цвет акцента (под
бренд салона).

### 24. PWA push-уведомления `S`

**Уже сделано** инфраструктурно — нужно UI для opt-in (toggle в Settings).

### 25. Импорт из Google Sheets `S`

Помимо CSV — drag&drop URL Google Sheet → парсинг → импорт. Многие салоны ведут
учёт в Sheets, миграция к нам легче.

### 26. WhatsApp шаблоны как UI `M`

Library pre-approved WA-templates с переменными. После Meta App Review мы
сможем слать клиентам WhatsApp без 24-h window — мощно для напоминаний.

### 27. Печать чеков для клиента `S`

Кнопка «распечатать» в визите → красивый чек с логотипом, услугами, суммой,
QR-кодом для отзыва. Физический эквивалент receipt.

## 🛠 Infra / Tech debt (для устойчивого роста)

### 28. Auto-refresh long-lived tokens `S`

FB/IG tokens ~60 дней — нужен cron, который за 7 дней до истечения
автоматически refresh'ит. Без этого через 60 дней все интеграции отвалятся.

### 29. Token rotation после Meta App Review `S`

Сейчас App Secrets засветились в чате — после Meta Review их можно сбросить и
ротировать без downtime (есть один backup token endpoint).

### 30. Sentry production setup + alerts `S`

Конфиг есть, нужно настроить алерты на error rate &gt; 1% за 5 мин → Telegram
владельцу.

### 31. CI E2E против staging `M`

Playwright на критические user-flow (signup → onboarding → first visit). Гоняем
на каждом PR на staging-проекте — ловим регрессии до prod.

### 32. Backup/restore plan `S`

Document + cron на еженедельный backup Supabase (через point-in-time recovery
API). После 5 клиентов это критично.
