# 05. UX Flows

## Принципы UX

1. **Mobile-first.** Владелица работает за креслом и смотрит в телефон.
2. **Минимум полей.** Спрашиваем только то, без чего нельзя обойтись.
3. **3 клика до результата.** Любая основная задача — максимум 3 клика.
4. **Empty states продают.** "Добавьте первый визит и увидите магию", не "нет данных".
5. **Skeleton loading везде.** Никаких full-page спиннеров.
6. **Optimistic updates.** Через TanStack Query.
7. **Toast-уведомления.** Не alert.
8. **Тон коммуникации:** на "ты", по-домашнему.

## Дизайн-система

**Источник истины — Hi-fi прототип в `Design/`.** Палитра, типографика, layout каждого ключевого экрана определены там и подняты в код через `apps/web/src/styles/globals.css` + `apps/web/tailwind.config.ts`. Подробности в [`decisions/007-design-tokens.md`](../decisions/007-design-tokens.md).

Краткая палитра (полная — в ADR-007):

- **navy** `#1A1A2E` — primary, активный sidebar, заголовки
- **teal** `#1E6B8A` — secondary, ссылки, AI-блок
- **sage** `#2E9E6B` — прибыль, рост
- **red** `#C0392B` — расходы, ошибки
- **yellow** `#FFF9C4` — подсветка инпута сумм
- **gold** `#C9A24B` — Pro-план в sidebar

**Шрифты:** Plus Jakarta Sans (UI), Inter (fallback), JetBrains Mono с tabular figures для всех чисел (класс `.num` или `font-mono`). Все через `@fontsource/*`, без Google Fonts CDN.

**Валюта** в UI берётся из `salons.currency` через `lib/format-currency.ts`. В прототипе показан €, но для каждого салона символ свой (PLN/EUR/USD). Никогда не зашиваем символ.

## Главные флоу

### Flow 1 — Регистрация → первый визит (целевое: 5 минут)

Онбординг — 5 шагов, пиксельный референс: `Design/project/screens-4-5-mobile.jsx` → `OnboardingScreen` (виден шаг 2 «Мастера»).

```
[Лендинг] → "Начать бесплатно" → [/signup] → email confirm → [/auth/callback]
   │
   ▼
[/onboarding]
   Шаг 1 «Салон»     — имя, страна (autoset валюта/timezone), тип салона
   Шаг 2 «Мастера»   — карточки мастеров с именем, специализацией, схемой оплаты
                       (стадия 1: только % от выручки; полный payout_scheme — TASK-21)
   Шаг 3 «Услуги»    — добавить стартовые услуги (или принять seed по типу салона)
   Шаг 4 «Расходы»   — назвать дефолтные категории (Аренда/Зарплата/Материалы/Реклама/...)
   Шаг 5 «Готово»    — кнопка «Открыть дашборд»
   │
   ▼ Edge function create-salon (атомарно: salon + member + categories + services + staff)
[/{salonId}/dashboard]
   │ Empty state «Добавь первый визит, чтобы увидеть магию»
   ▼ FAB «+ Визит»
[QuickEntryModal — см. Flow 2]
   │ Аня · Маникюр гель · €40 · Карта → Сохранить
   ▼ Toast «Визит добавлен · Откатить»
[/{salonId}/dashboard]
   Прибыль обновилась
```

### Flow 2 — Ввод визита с улицы (3 клика)

Пиксельный референс: `Design/project/screens-2-3.jsx` → `QuickEntryModal`.

```
[Любая страница] — FAB «+ Визит» снизу справа
   │
   ▼
[Modal «Новый визит»]
   Дата:        read-only «6 мая 2026, понедельник» + pill «сегодня»
   Мастер:      select с цветным аватаром (по умолчанию — последний)
   Услуга:      typeahead, рядом подсказка ≈ €40 (default_price из БД)
   Сумма:       большой жёлтый инпут, € слева, JetBrains Mono, font-size 32px
                — самый заметный элемент модалки
   Оплата:      pills под суммой [Наличные][Карта][Перевод] (по умолчанию — Карта)
   Комментарий: маленький инпут (опц.)
   ─────────────────────────────────────
   [   Сохранить визит   ]   ← navy, full-width
   Сохранить и добавить ещё  ← teal-link, для сценария «несколько подряд»
   │
   ▼ Сохранить (optimistic update через TanStack Query)
[Toast: ✓ Визит добавлен · Откатить]
```

## Главные экраны

Пиксельный референс — `Design/`. Здесь — только намерение и поведение, без вайрфреймов.

### Dashboard (`/{salonId}/dashboard`)

Референс: `Design/project/screen-dashboard.jsx` (desktop), `screens-4-5-mobile.jsx` → `MobileDashboard` (mobile).

**Иерархия (что бросается в глаза первым):**

1. **Прибыль** — самая большая карточка, тёмно-navy с sage-glow в углу, шрифт 48px JetBrains Mono. Это «главное число» — UTP продукта.
2. **Выручка / Расходы** — две меньших карточки слева, sage и red акценты соответственно.
3. **Выручка по мастерам** — горизонтальные progress-bars с аватарами и сервисами мастера.
4. **Тип оплаты** — donut chart (Наличные / Карта / Перевод) с центральной суммой.
5. **Топ услуг месяца** — ряд из 5 карточек: маржа цветной точкой (sage/gold/red по уровню), название, выручка, кол-во визитов.
6. **Последние записи** — таблица 5 строк: дата, мастер, услуга, сумма (sage), pill оплаты.

**День-чарт по выручке** — НЕ на дашборде, переехал в TASK-23 «Аналитика».

**Mobile** (см. `MobileDashboard`):

- Прибыль-карточка первая, на весь экран
- Выручка/Расходы — компактный 2-колоночный grid под ней
- Master bars — без аватаров и подписей сервисов (компакт)
- «Сегодня» — список последних 2 визитов
- FAB круглый сверху bottom-nav
- Bottom-nav: Главная / Визиты / Расходы / AI / Ещё

### Расходы (`/{salonId}/expenses`)

Референс: `Design/project/screens-2-3.jsx` → `ExpensesScreen`.

**Layout:**

- Заголовок «Расходы» + сумма за период (red, mono)
- Кнопка «+ Добавить расход» (teal, не FAB)
- 4 категории-карточки в ряд (border-left цветной): Аренда / Зарплата / Материалы / Реклама — суммы по каждой
- 2 колонки:
  - **Список расходов** (таблица): дата, иконка категории, название + категория, сумма (red), редактировать
  - **Структура расходов** (progress-bars): категория, сумма + %, заполненная полоса цветом категории
- Карточка **AI-подсказки** (только декоративный плейсхолдер в стадии 1, реальный AI-движок — TASK-33; в стадии 1 показываем только если правила от стадии 4 уже есть, иначе скрываем)

## Empty states

**Дашборд (нет данных):**

```
  📊
  Здесь появится твой
  дашборд прибыли

  Добавь первый визит,
  чтобы увидеть магию.

  [  + Добавить визит  ]
```

**Список визитов:** `Пока нет визитов. [+ Добавить первый визит]`

**Расходы:** `Аренда, материалы, реклама — всё, что ты тратишь. [+ Добавить расход]`

## Loading / Error states

- **Skeleton** для карточек и списков
- **Spinner маленький** в кнопках при submit
- **Никогда** full-screen overlay
- **Toast** для transient errors
- **Inline error** под полем формы (Zod messages)
- **Empty state с Retry** для page-level errors

## Уведомления

| Тип            | Где                 | Когда                             |
| -------------- | ------------------- | --------------------------------- |
| Toast зелёный  | Снизу               | Успешное действие                 |
| Toast красный  | Снизу               | Ошибка                            |
| Баннер жёлтый  | Сверху страницы     | Триал кончается, оплата не прошла |
| Баннер красный | Сверху              | Подписка истекла, read-only       |
| Email          | Postmark            | Welcome, billing, weekly digest   |
| Push           | Web Push (стадия 4) | Новый инсайт, конец триала        |

## Адаптивность

- **Desktop (≥1024px):** sidebar слева, side panels для деталей
- **Tablet (≥768px):** desktop layout, компактнее
- **Mobile (<768px):** bottom nav, drawer для sidebar, bottom sheets для форм

## Доступность

- `focus-visible` рамка на всех интерактивных элементах
- Контраст ≥ 4.5:1 (WCAG AA)
- ARIA labels на иконках без текста
- Все формы работают с клавиатуры
- shadcn/ui это даёт из коробки
