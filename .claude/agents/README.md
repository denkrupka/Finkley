# Claude Code Sub-Agents

Каждый файл `.md` в этой папке — это **custom sub-agent** Claude Code. Формат frontmatter:

```yaml
---
name: agent-name
description: Когда главный агент должен вызвать этого suba-agent
tools: Read, Grep, Glob # белый список инструментов
model: opus # opus | sonnet | haiku | inherit
---
<system prompt с ролью и правилами>
```

Поля `tools` и `model` опциональны: без `tools` агент наследует все инструменты главного треда (изоляция теряется), без `model` использует модель по умолчанию.

## Зачем нужны sub-agents

Когда работаешь над крупной задачей, основной агент держит весь контекст: задачу, код, тесты, миграции, ADR. Это **загрязняет** его внимание.

Sub-agents позволяют делегировать узкие задачи в отдельный изолированный контекст:

- `architect` — обсудить архитектурное решение, написать ADR, не отвлекаясь на код
- `migrator` — только SQL-миграции, не лезть в код приложения
- `reviewer` — посмотреть на готовый PR "холодным взглядом", без истории дискуссий
- `tester` — написать тесты для существующего кода

Главный агент **вызывает** sub-agent → тот делает задачу в чистом контексте → возвращает результат.

## Как использовать (если работает)

В Claude Code:

> "Используй sub-agent `reviewer` чтобы посмотреть на этот PR"
> "Передай задачу `migrator` — нужна миграция для добавления поля X в таблицу Y"

Главный агент сам делегирует и принимает результат.

## Файлы в этой папке

- `architect.md` — для крупных архитектурных обсуждений и ADR
- `migrator.md` — только для SQL-миграций БД
- `reviewer.md` — для код-ревью PR
- `tester.md` — для написания тестов

Каждый файл — это **system prompt** для соответствующего sub-agent с его узкой специализацией.
