# ADR-006: Stripe + Stripe Tax для платежей

## Статус

`Accepted` · 2026-05-05

## Контекст

Нужен платёжный провайдер для подписочной модели €15/мес с поддержкой:

- VAT для разных стран EU (PL/DE/LT/EE/CZ — наш ICP)
- Триал 14 дней без карты
- Money-back guarantee 7 дней
- Customer self-service (отмена, смена карты, скачивание инвойсов)
- Минимум интеграционной работы для соло-разработчика

## Решение

**Stripe + Stripe Tax + Stripe Customer Portal.**

- **Stripe Checkout** (хостед страница) — не пишем свой UI ввода карты. Меньше PCI-compliance забот.
- **Stripe Tax** — автоматический расчёт VAT по стране покупателя. У нас юрлицо в PL, регистрируемся как VAT MOSS / OSS. Stripe собирает VAT с EU-клиентов и складывает к €15 = €18.45 для DE, €18.45 для PL, и т.д.
- **Customer Portal** — хостед страница, где юзер сам меняет карту, отменяет, скачивает инвойсы. Не пишем свой UI.
- **Webhooks** — наш единственный backend-код по платежам (edge function `stripe-webhook`).
- **Прайс:** €15/мес brutto в RU-локализации мы говорим "€15/мес (с НДС)", в Stripe Product создаём цену 15.00 EUR с `tax_behavior=inclusive` ИЛИ `exclusive` — выбираем `exclusive` (Stripe Tax добавляет VAT сверху, юзер видит итоговую сумму на checkout).

> Окончательное решение про inclusive/exclusive — на этапе TASK-16, после консультации с польским бухгалтером (это влияет на учёт и фискализацию).

## Альтернативы

- **Paddle.** Они — Merchant of Record, сами платят VAT и нам деньги приходят netto. Привлекательно, но: высокая комиссия (~5–8% против Stripe ~1.5+%), меньше гибкости, исторически с проблемами для CIS-юзеров. **Резерв на план Б** если Stripe вдруг блокирует аккаунт.
- **Lemon Squeezy.** Тоже MoR, но молодой сервис. Резерв.
- **Свой биллинг** — отклонено, мы соло-разработчик.

## Последствия

### Положительные

- Минимум кода: 2 edge functions (`create-checkout-session`, `stripe-webhook`) и Customer Portal ссылка
- VAT не наша проблема (Stripe Tax автоматизирует)
- PCI-compliance — Stripe берёт на себя
- Stripe документация и примеры — лучшие в индустрии
- Subscriptions + trials + dunning (повторные попытки оплаты) встроены

### Отрицательные

- Зависимость от одного вендора. **Митигация:** план Б (Paddle) задокументирован, миграция возможна за 1-2 дня (миграция требует выгрузки customer data из Stripe, по их API доступно)
- Stripe иногда блокирует аккаунты бьюти-индустрии без объяснений (хотя нам как SaaS не должно касаться, но был precedent). **Митигация:** чётко описать в Stripe profile "B2B SaaS for beauty salons accounting", не использовать слова типа "massage" в product name
- Комиссия 1.5% + €0.25 на транзакцию = ~€0.50 с €15 платежа = ~3.3% эффективной комиссии. **Принимаем как cost of doing business.**

### Что мониторим

- Failed payment rate (если >5% — пересмотреть dunning стратегию)
- Stripe Tax: правильность VAT расчётов в первый месяц
- Если приходит письмо от Stripe о возможной блокировке — немедленно реагируем
