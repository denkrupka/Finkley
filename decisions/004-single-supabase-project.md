# ADR-004: Один Supabase проект, не несколько

## Статус

`Accepted` · 2026-05-05

## Контекст

Возможные стратегии для multi-tenant SaaS:

1. **Один Supabase проект, разделение через RLS** — все данные в одной БД, один URL и один anon-key для всех юзеров.
2. **Проект на тенанта** — для каждого крупного клиента отдельный Supabase проект (database-per-tenant).
3. **Гибрид:** один проект для основных данных, отдельный — для агрегатов/аналитики.

## Решение

**Один Supabase проект (вариант 1)** для production. Отдельный проект — для staging.

Multi-tenancy реализуется через:

- Колонка `salon_id` на каждой таблице с пользовательскими данными
- RLS-политики, проверяющие членство пользователя в `salon_members`
- Все запросы из браузера идут с anon-key, RLS защищает

## Альтернативы

- **Database-per-tenant.** Отклонено: огромная операционная нагрузка для соло-разработчика (управление N БД, миграции на каждой, мониторинг). Имеет смысл при enterprise-клиентах с регуляторными требованиями (B2B health/finance), нам не нужно.
- **Гибрид.** Отклонено: преждевременная оптимизация. Если упрёмся в лимиты Supabase или появится потребность в анонимизированных бенчмарках с другим life cycle — мигрируем тогда.

## Последствия

### Положительные

- Минимум операционки: одни миграции, один dashboard, один backup
- RLS-тестов хватает для проверки изоляции
- Стандартный паттерн для SaaS, много примеров в сообществе

### Отрицательные

- Один баг в RLS-политике = потенциальная утечка между тенантами. **Митигация:** автоматизированные RLS-тесты на CI для каждой таблицы.
- Нельзя дать одному клиенту "свой регион" БД (для ENT — но мы пока не enterprise).
- Limit на размер БД Supabase free tier — 500MB. **Митигация:** при росте — Supabase Pro ($25/мес, 8GB) или партиционирование старых данных.

### Что мониторим

- Размер БД (предупреждение при 70% от лимита)
- Кол-во ошибок RLS в Sentry — если RLS падает, юзер увидит "0 строк", это хороший сигнал но плохой UX
- Если придёт enterprise-клиент с требованием изоляции — сделать ему отдельный Supabase проект
