# ADR-035: Годовой биллинг −15%, цены всегда «/мес», дефолт годовой

## Статус

`Accepted`

Дата: 2026-06-26

## Контекст

В 6-уровневой тарифной модели (ADR-033) была только месячная оплата. Чтобы
повысить LTV/удержание и снизить отток, владелец захотел предложить годовую
оплату со скидкой **−15%** ко всем платным планам. Требование к подаче: цены
**везде показываются «/мес»** (чтобы число не «прыгало» в порядок раз больше),
а годовой режим показывает **скидочную цену в месяц** + подпись «при оплате за
год». По умолчанию выбран **годовой** интервал (юзер сразу видит самую низкую
цену /мес), с возможностью переключиться на «Месяц» и увидеть полные цены.

Переключатель нужен на двух поверхностях: страница тарифов лендинга (Astro,
статика) и биллинг-пикер приложения (`BillingButtons.tsx`).

## Решение

**Скидка** — множитель `0.85` (−15%) к полной месячной цене. Точные числа:

| План | Месяц (полный) | Год: цена /мес (×0.85) | Год: платёж за год (×12×0.85) |
| ---- | -------------- | ---------------------- | ----------------------------- |
| t19  | €19            | €16,15                 | €193,80                       |
| t49  | €49            | €41,65                 | €499,80                       |
| t69  | €69            | €58,65                 | €703,80                       |
| t99  | €99            | €84,15                 | €1009,80                      |

Десятичные показываем только при наличии дроби (19 → «19»; 16,15 → «16,15»).
Разделитель — по локали через `Intl.NumberFormat` (ru/pl: запятая; en: точка).

**Приложение** (`apps/web/src/routes/billing/BillingButtons.tsx`):

- Состояние `billingInterval: 'month' | 'year'`, дефолт `'year'`.
- Сегмент-переключатель `IntervalToggle` над грид-пикером: «Год · −15%» /
  «Месяц», активная кнопка — `bg-brand-navy`.
- `PlanCard` рендерит цену через `formatMonthlyPrice(PLAN_PRICE_EUR[plan], interval, locale)`;
  при `year` показывает мелкую подпись «при оплате за год».
- `startCheckout` передаёт `interval` в `create-checkout-session`.
- Чистый helper `apps/web/src/lib/billing-interval.ts` (+ unit-тесты).

**Лендинг** (`apps/landing/src/components/PricingBody.astro` +
`apps/landing/src/i18n/content/pricing.ts`):

- Лейблы интервала в `PricingCopy.interval` (ru + pl).
- Переключатель над карточками платных тарифов, дефолт «Год».
- Каждая платная карточка несёт `data-monthly` (число); inline `<script>`
  (vanilla JS, т.к. Astro статичен) пересчитывает отображаемую цену (×0.85 при
  year), форматирует по `data-locale` и показывает/прячет подпись «при оплате за
  год». Free/Demo карточки переключатель не трогает.

**Stripe backend**:

- `supabase/functions/_shared/plans.ts`: добавлен годовой маппинг env
  `STRIPE_PRICE_T19_ANNUAL` / `_T49_ANNUAL` / `_T69_ANNUAL` / `_T99_ANNUAL`.
  `priceIdForPlan(plan, interval = 'month')` берёт annual price при `year`.
  `planForPriceId` (webhook) матчит и месячные, и годовые price id → тот же plan.
- `create-checkout-session/index.ts` принимает `interval` в body (валидируется,
  дефолт `'month'`). Если годовая цена не настроена (env пуст) — graceful 500
  `price_not_configured` (как для месячной).

## Альтернативы, которые рассматривали

- **Показывать годовую сумму целиком (€193,80/год):** отклонено — число резко
  крупнее месячного, сложнее сравнивать планы. Владелец выбрал «всегда /мес».
- **Скидка в процентах, считаем на лету в Stripe (coupon):** отклонено —
  годовая цена должна быть отдельным Stripe price (interval=year), это чище для
  инвойсов и пропорций при смене плана через Customer Portal.
- **Один price с переключением quantity:** не подходит для подписок с разным
  биллинг-интервалом.

## Последствия

### Положительные

- Выше LTV/удержание за счёт годовых подписок.
- Единая подача «/мес» — число не прыгает, проще сравнивать планы.
- Документация/API планов не расходятся: helper + реестр env.

### Отрицательные

- Владельцу нужно **создать 4 новые годовые цены в Stripe** (interval=year) и
  прописать env `STRIPE_PRICE_T*_ANNUAL`. Пока они не настроены — выбор «Год»
  для соответствующего плана отдаст `price_not_configured` (graceful 500),
  месячная оплата работает как раньше.
- Числа скидки (×0.85) и round-to-2 продублированы в TS-helper и landing-скрипте
  (Astro-скрипт не импортирует web-пакет) — менять надо в обоих местах.

### Что мониторим

- Если доля годовых подписок низкая или растёт отток на ренью — пересмотреть
  размер скидки (только владелец).
- Если Stripe Tax/инвойсы покажут расхождение центов на годовом платеже —
  свериться с точными суммами из таблицы выше.

## Что делает владелец в Stripe

1. Для каждого платного плана создать **годовую цену** (Recurring, interval =
   Yearly) на сумму годового платежа из таблицы: €193,80 / €499,80 / €703,80 /
   €1009,80.
2. Скопировать `price_…` id каждой и прописать в секреты Supabase:
   `STRIPE_PRICE_T19_ANNUAL`, `STRIPE_PRICE_T49_ANNUAL`,
   `STRIPE_PRICE_T69_ANNUAL`, `STRIPE_PRICE_T99_ANNUAL`.
3. Месячные цены/секреты (`STRIPE_PRICE_T19…`) остаются без изменений.
